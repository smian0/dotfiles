## Adversarial Research Findings

### Common Pain Points & Limitations

#### From Search: "Xero bank register overpayment problems"

**Documented Issues:**
- **UI Confusion:** Users report difficulty finding overpayment options if transaction not properly matched first
- **Amount Threshold:** Overpayment detection may not trigger for very small discrepancies (under currency threshold)
- **Multi-currency:** Additional UI complexity when dealing with foreign currency overpayments
- **Refund Timing:** Confusion about when refund transaction actually processes vs. records in system

#### From Search: "Xero overpayment reconciliation errors"

**Common Errors:**
- **"Cannot reconcile - amount mismatch":** Occurs when trying to force-match overpayment without selecting proper option
- **Bank Feed Delays:** Imported transactions may not immediately show overpayment options until fully synced
- **Permission Issues:** Some users lack permissions to create credit notes or refunds, limiting overpayment handling options
- **Reconciliation State:** Overpayment options disappear if user navigates away before completing reconciliation

**Mitigation Strategies:**
- Always complete the match process before expecting overpayment options
- Check user permissions for credit note and refund creation
- Wait for bank feed sync to complete before reconciling
- Don't navigate away from reconciliation screen until process is complete

## Detailed UI Workflow

### Primary Path for Purchase Invoice Overpayment in Bank Register:

**Step-by-Step Navigation:**

1. **Navigate to Bank Register:**
   - Click **Banking** in main navigation
   - Select **Bank Accounts**
   - Choose the relevant bank account
   - Click **Reconcile** tab

2. **Locate Overpaid Transaction:**
   - Find the transaction in the unreconciled transactions list (left panel)
   - Transaction will show the actual bank amount
   - Use search/filter if needed to locate specific transaction

3. **Initiate Match:**
   - Click **Find & Match** button
   - Search for the corresponding purchase invoice/bill
   - Xero may auto-suggest matches based on amount and date

4. **Select Invoice Match:**
   - Click on the purchase invoice to match
   - UI displays **invoice amount** vs. **bank transaction amount**
   - Amount discrepancy triggers overpayment detection

5. **Overpayment Options Appear:**
   - Xero automatically detects excess amount
   - Displays three options (typically as dropdown or radio buttons):
     - **Create credit note** (for future invoice allocation)
     - **Allocate to prepayment** (creates prepayment balance)
     - **Refund overpayment** (initiates refund process)

6. **Select Handling Method:**
   - Choose appropriate option based on business need
   - For purchase invoices to suppliers:
     - **Credit note** = Apply to future bills from this supplier
     - **Prepayment** = Hold as balance on supplier account
     - **Refund** = Request money back from supplier

7. **Complete Reconciliation:**
   - Click **OK** or **Create** button to finalize
   - Xero processes the reconciliation and records overpayment

8. **Verify Results:**
   - Check bank register shows transaction as reconciled
   - Verify supplier account reflects chosen overpayment treatment
   - Review created credit note/prepayment/refund transaction

### UI Elements Reference:

**Main Interface Components:**
- **Left Panel:** Unreconciled transactions from bank feed
- **Right Panel:** Reconciliation details and matching interface
- **Reconciliation Panel:** Shows match details, amounts, and options
- **Find & Match Button:** Opens invoice search interface
- **Amount Comparison Display:** Visual indicator showing invoice vs. transaction amount
- **Overpayment Badge/Label:** Icon or text indicating overpayment detected
- **Options Dropdown/Radio Buttons:** Selection for overpayment handling method
- **OK/Create/Reconcile Button:** Final confirmation to complete reconciliation

**Visual Indicators:**
- Amount discrepancy highlighted in red or with warning icon
- Overpayment amount shown separately
- Matched invoice displays with checkmark or highlight
- Unreconciled transactions have distinct visual state

## Key Takeaways

### Most Important Findings:

1. **Automatic Detection:** Xero automatically detects overpayments during reconciliation - no manual flagging needed

2. **Three Standard Options:** Credit note, prepayment allocation, or refund are consistently available across all Xero accounts

3. **Match First, Options Second:** Overpayment handling options only appear AFTER matching transaction to invoice - this is the most common source of user confusion

4. **Purchase Invoice Specific:** The workflow for supplier overpayments (purchase invoices) follows the same pattern as customer overpayments but appears in supplier account instead

5. **UI Consistency:** The reconciliation panel structure is consistent across all transaction types, making it easier to learn once and apply everywhere

### Common Mistakes to Avoid:

- Attempting to select overpayment option before completing invoice match
- Navigating away from reconciliation screen before finalizing
- Not waiting for bank feed sync to complete
- Assuming options will appear without proper user permissions
